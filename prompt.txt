1. Create src/news_collectors/base.py

Define a NewsItem dataclass with fields:
- news_id: str
- source: str
- title: str
- url: str | None
- published_at_utc: str | None
- retrieved_at_utc: str
- raw_summary: str | None

2. Create src/news_collectors/dedupe.py

Implement deterministic deduplication helpers:
- canonicalize_url(url: str | None) -> str | None
- normalize_title(title: str) -> str
- hash_text(value: str) -> str
- compute_payload_hash(payload: dict) -> str
- assign_duplicate_group_ids(items: list[NewsItem]) -> list[dict]

Rules:
- URL canonicalization should remove common tracking query parameters such as utm_source, utm_medium, utm_campaign, utm_term, utm_content.
- Normalize titles by lowercasing, stripping whitespace, collapsing repeated spaces, and removing simple punctuation.
- Duplicate grouping should prefer canonical_url when available; otherwise use normalized_title_hash.
- Do not use fuzzy matching yet.

3. Create src/news_collectors/rss_collector.py

Implement a lightweight RSS collector using feedparser.

Functions:
- collect_from_feed(feed_url: str, source_name: str) -> list[NewsItem]
- collect_from_feeds(feeds: list[dict]) -> list[NewsItem]

Each feed config dict should include:
- source
- url

Rules:
- Store only metadata and RSS-provided summary/snippet.
- Do not fetch full article body.
- Do not scrape article pages.
- Convert timestamps to UTC using existing time_utils where practical.
- If published timestamp is missing or ambiguous, store None and continue.
- retrieved_at_utc should be current UTC time.
- collection_mode should be "rss".
