Research question:
Do overnight U.S. semiconductor / AI news signals provide incremental explanatory power for Taiwan semiconductor / AI supply-chain returns after controlling for U.S. market and sector returns?

Goal:
Create a validation layer that compares baseline-only models against news+baseline models using time-based train/test splits.

Inputs:
- data/processed/research_panel_<label>.csv

Outputs:
- data/processed/oos_validation_results_<label>.csv
- data/reports/oos_validation_summary_<label>.md

Key principles:
- Splits must be chronological by taiwan_trading_date.
- No random shuffle split.
- If sample size is insufficient, return status = insufficient_sample.
- Do not infer causality.
- Do not make trading recommendations.

Default dependent variables:
- prev_close_to_open_return
- open_to_close_return
- close_to_close_return
- next_close_to_close_return

Default news features:
- sum_impact_score
- mean_impact_score
- max_abs_impact_score
- mean_combined_confidence
- candidate_count
- is_potentially_positive
- is_potentially_negative

Default baseline controls:
- qqq_return
- soxx_return
- smh_return
- nvda_return
- amd_return
- tsm_adr_return

Model families:
1. baseline_only:
   y ~ qqq_return + soxx_return + smh_return + nvda_return + amd_return + tsm_adr_return

2. news_only:
   y ~ sum_impact_score + mean_combined_confidence + candidate_count

3. news_plus_baseline:
   y ~ sum_impact_score + mean_combined_confidence + candidate_count
       + qqq_return + soxx_return + smh_return + nvda_return + amd_return + tsm_adr_return

4. directional_plus_baseline:
   y ~ is_potentially_positive + is_potentially_negative
       + qqq_return + soxx_return + smh_return + nvda_return + amd_return + tsm_adr_return

Implementation requirements:

1. Create src/backtest/oos_validation.py

Implement:
- validate_oos_inputs(panel_df) -> dict
- chronological_train_test_split(panel_df, test_fraction=0.3, min_train_dates=20, min_test_dates=10) -> tuple[pd.DataFrame, pd.DataFrame, dict]
- fit_train_evaluate_test(train_df, test_df, y_col, x_cols, min_train_obs=30, min_test_obs=10) -> dict
- run_oos_validation(panel_df, dependent_vars=None, model_specs=None, test_fraction=0.3) -> pd.DataFrame

Rules:
- Sort by taiwan_trading_date.
- Split by unique dates, not by rows.
- If unique dates < min_train_dates + min_test_dates, return insufficient_sample.
- Drop rows with NaN in required y/x columns per model.
- Fit OLS on train only.
- Predict on test.
- Compute:
  - train_n
  - test_n
  - train_r_squared
  - test_mse
  - test_mae
  - test_direction_accuracy for sign prediction:
      sign(predicted_return) == sign(actual_return)
  - test_mean_actual_return
  - test_mean_predicted_return
  - coefficient and t-stat for news variables if available
  - status
  - notes
