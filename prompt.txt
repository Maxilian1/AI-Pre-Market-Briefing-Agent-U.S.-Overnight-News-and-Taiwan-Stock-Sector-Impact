Aggregation logic:
Group by:
- taiwan_trading_date
- return_target
- return_target_type
- taiwan_target_type
- taiwan_sector

Within each group compute:
- candidate_count
- positive_count
- negative_count
- neutral_count
- unavailable_count
- mean_impact_score
- sum_impact_score
- max_abs_impact_score
- mean_combined_confidence
- final_directional_label

final_directional_label rules:
- If return_data_available is False for the group, final_directional_label = unavailable.
- Else if sum_impact_score > 0, final_directional_label = potentially_positive.
- Else if sum_impact_score < 0, final_directional_label = potentially_negative.
- Else if sum_impact_score == 0 and neutral_count > 0, final_directional_label = neutral.
- Else final_directional_label = neutral.

Preserve returns by taking the first non-null value per target/date:
- prev_close_to_open_return
- open_to_close_return
- close_to_close_return
- next_close_to_close_return

Only rows with return_data_available=True should be used for return-performance statistics.
Unavailable rows should be counted and reported separately.

3. T-stat behavior

compute_simple_t_stat(values):
- Drop NaN.
- If n < 2 or std == 0, return NaN for t-stat and include note.
- Else t = mean / (std / sqrt(n)).
- No p-value required in Phase 6B unless easy and lightweight.
